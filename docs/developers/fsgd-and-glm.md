# FSGD & GLM

## Excel → FSGD classification

`src/utils/excel_to_fsgd.py` converts the patient spreadsheet into a FreeSurfer
Group Descriptor. It classifies each variable column (index 3+):

- **all-numeric** column → **continuous** variable (an FSGD `Variables` entry / GLM
  covariate);
- column with **any non-numeric** value → **discrete factor**. Multiple discrete
  factors are folded into compound class names via a Cartesian product (e.g.
  `AD-F`).

The FSGD `Variables` line does **not** distinguish discrete from continuous —
FreeSurfer treats every entry as a numeric regressor.

## DODS vs. DOSS

Two designs are supported (passed to `mri_glmfit` and used to size the contrast
matrices):

- **DODS** — *Different Offset, Different Slope*: separate intercept + slope per
  group.

    $$n_{\text{regressors}} = n_{\text{classes}} \times (n_{\text{variables}} + 1)$$

- **DOSS** — *Different Offset, Same Slope*: pooled slopes across groups.

    $$n_{\text{regressors}} = n_{\text{classes}} + n_{\text{variables}}$$

Every `.mtx` row must have exactly `n_regressors` columns for the chosen design.
See [DODS vs DOSS](https://surfer.nmr.mgh.harvard.edu/fswiki/DodsDoss).

## Contrast semantics

- The contrast matrix is applied during **inference only**, not during
  $\hat{\beta}$ estimation.
- **Zeroing** a column in the contrast controls for a variable without testing it;
  **dropping** a regressor entirely is not equivalent.

## Auto-generation & the marker line

`run_analysis.py` auto-generates the FSGD when none is provided, writing it to
`analysis.fsgd` and ending it with a marker line
(`# Generated by run_analysis.py`). On re-runs it detects that marker and
regenerates the file so it stays in sync with the Excel input. A user-provided
FSGD (without the marker) is used as-is.
